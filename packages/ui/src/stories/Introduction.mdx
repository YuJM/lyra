# Lyra UI

**OpenAI Apps SDK 준수 컴포넌트 라이브러리**

Lyra UI는 [Base UI](https://base-ui.com/)를 기반으로 한 접근 가능하고 커스터마이징 가능한 React 컴포넌트 라이브러리입니다.
모든 컴포넌트는 OpenAI Apps SDK 디자인 가이드라인과 WCAG 2.1 Level AA를 준수합니다.

---

## 🎯 핵심 원칙

### 1. OpenAI Apps SDK 준수

Lyra UI는 ChatGPT와 같은 AI 애플리케이션에 최적화되어 있습니다.

**5대 원칙**:
- **Conversational (대화형)**: 자연스러운 대화 흐름 지원
- **Intelligent (지능형)**: 컨텍스트 기반 인터랙션
- **Simple (단순함)**: 최소한의 UI, 명확한 계층
- **Responsive (반응형)**: 빠른 피드백, 스트리밍 지원
- **Accessible (접근성)**: WCAG 2.1 AA 100% 준수

📖 [OpenAI 가이드라인 보기](/story/guidelines-openai--docs)

---

### 2. 접근성 우선 (Accessibility First)

모든 컴포넌트는 다음을 보장합니다:

- ✅ **WCAG 2.1 Level AA** 100% 준수
- ✅ **키보드 네비게이션** 완전 지원
- ✅ **스크린 리더** 최적화
- ✅ **색상 대비율** 최소 4.5:1 (일반 텍스트), 3:1 (UI)
- ✅ **포커스 관리** 명확한 인디케이터

📖 [접근성 가이드 보기](/story/guidelines-accessibility--docs)

---

### 3. 디자인 토큰 기반

일관된 디자인 시스템을 위해 CSS 변수를 사용합니다.

```css
/* 예시: 버튼 스타일 */
button {
  background: var(--color-primary);
  color: var(--color-on-primary);
  border-radius: var(--border-radius-md);
  padding: var(--spacing-2) var(--spacing-4);
}
```

## 🚀 빠른 시작

### 설치

```bash
pnpm add @lyra/ui
```

### 사용법

```tsx
import { Button, Dialog } from '@lyra/ui';
import '@lyra/ui/styles';

function App() {
  return (
    <Dialog.Root>
      <Dialog.Trigger asChild>
        <Button>대화 상자 열기</Button>
      </Dialog.Trigger>
      <Dialog.Portal>
        <Dialog.Backdrop />
        <Dialog.Popup>
          <Dialog.Title>제목</Dialog.Title>
          <Dialog.Description>설명</Dialog.Description>
          <Dialog.Close asChild>
            <Button>닫기</Button>
          </Dialog.Close>
        </Dialog.Popup>
      </Dialog.Portal>
    </Dialog.Root>
  );
}
```

## 🛠️ 개발 가이드

### 컴포넌트 개발 체크리스트

새로운 컴포넌트를 개발할 때:

#### 필수 항목
- [ ] Base UI 기반 구현 (없으면 Headless UI 고려)
- [ ] TypeScript 타입 완전 정의
- [ ] WCAG 2.1 AA 준수 (Storybook A11y 탭에서 0개 위반)
- [ ] 키보드 네비게이션 지원
- [ ] 다크모드 지원
- [ ] Storybook 스토리 작성 (최소 3개: Default, Variants, Interactive)

#### 권장 항목
- [ ] 단위 테스트 (Vitest)
- [ ] 시각적 회귀 테스트 (Chromatic)
- [ ] 성능 최적화 (React.memo, useMemo 등)
- [ ] 문서화 (JSDoc, MDX)

📖 [접근성 체크리스트 보기](https://github.com/lyra/lyra/blob/main/claudedocs/wcag-2.1-aa-checklist.md)

---

### 테스트 방법

#### 1. 자동 테스트

```bash
# 단위 테스트
pnpm test

# 접근성 테스트
pnpm test:a11y

# 시각적 회귀 테스트
pnpm chromatic
```

#### 2. 수동 테스트

**키보드 네비게이션**:
- Tab: 포커스 이동
- Enter/Space: 활성화
- Escape: 닫기
- Arrow keys: 리스트/그룹 네비게이션

**스크린 리더**:
- NVDA (Windows)
- VoiceOver (macOS)
- TalkBack (Android)

**브라우저 확대**:
- 200% 확대 시 레이아웃 유지
- 400% 확대 시 리플로우 확인
